SAMPLE PROMPT 1 — Family Medicine SOAP Note

Structured, complete, and ready to paste into your EHR after review.

⚠️ HIPAA NOTICE: Never enter real patient names, date of birth, SSN,
or any Protected Health Information (PHI) into AI tools.
Use [Patient A] or aliases. This prompt is for documentation drafting
assistance only — final clinical judgment remains with the provider.

You are a clinical documentation assistant helping a family medicine provider draft a SOAP note.

Fill in the details below using de-identified or fictional information:

Patient alias: [e.g., Patient A]
Age: [Age]
Sex: [Male / Female / Other]
Chief Complaint: [e.g., "sore throat and fever for 3 days"]
Visit type: [Acute sick visit / Follow-up / Wellness / Telehealth]

SUBJECTIVE:
- Onset: [When symptoms started]
- Location: [Where applicable]
- Duration: [How long]
- Character: [Quality of symptoms, e.g., "sharp, constant pain"]
- Aggravating/Relieving factors: [What makes it worse/better]
- Associated symptoms: [e.g., "fever 101.2, mild cough, no nausea"]
- Relevant history: [Relevant PMH, medications, allergies — use fictional data]
- Review of Systems pertinent positives/negatives: [List briefly]

OBJECTIVE:
- Vital signs: [T: / HR: / BP: / RR: / SpO2: / Weight if relevant]
- General appearance: [e.g., "Alert, oriented, mild distress"]
- Relevant exam findings by system: [List each system examined]
- Diagnostic results (if any): [e.g., "Rapid strep: positive" or "pending"]

ASSESSMENT:
- Primary diagnosis: [ICD-10 code + description, or description only]
- Secondary/differential diagnoses (if applicable): [List]

PLAN:
- Medications: [Name, dose, route, frequency, duration]
- Orders: [Labs, imaging, referrals]
- Patient education provided: [Topics discussed]
- Follow-up: [When and under what conditions]

Draft a complete, structured SOAP note using the information above.
Format each section clearly with standard clinical language.

SAMPLE PROMPT 2 — Prior Authorization Letter (Medication)

The letter that takes 45 minutes to write from scratch — structured to include what insurers actually require.

⚠️ HIPAA NOTICE: Never enter real patient names, date of birth, SSN,
or any Protected Health Information (PHI) into AI tools.
Use [Patient A] or aliases. This prompt is for documentation drafting
assistance only — final clinical judgment remains with the provider.

You are a clinical documentation assistant helping a provider write a prior authorization letter for a medication.

Details (use de-identified or fictional information):
- Patient alias: [e.g., Patient A]
- Age: [Age]
- Diagnosis (ICD-10 preferred): [e.g., "Major Depressive Disorder, recurrent, moderate — F33.1"]
- Medication requested: [Drug name, dose, route, frequency]
- Step therapy drugs tried (if applicable): [List each drug, dose, duration, reason for discontinuation — e.g., "Sertraline 200mg for 8 weeks — partial response / Escitalopram 20mg for 6 weeks — intolerable side effects"]
- Clinical justification: [Why this specific medication is medically necessary — e.g., "Patient failed two first-line agents. Prior authorization required for third-line agent. Clinical guidelines support escalation."]
- Relevant labs or diagnostics: [If applicable, e.g., "PHQ-9 score: 18 (moderate-severe)" or "none"]
- Ordering provider name & NPI: [Provider Name, NPI #]
- Practice/facility name: [Name]
- Insurance plan / payer: [Insurance name]

Write a prior authorization letter that includes:
1. Patient identification (alias) and diagnosis
2. Requested medication with full details
3. Step therapy history with clinical rationale for each failure
4. Medical necessity justification with clinical guideline references (cite general guideline, not proprietary)
5. Supporting clinical data (labs, scores, objective measures)
6. Request for expedited review if clinically urgent (optional — specify if needed)
7. Provider signature block

Format: Formal letter. Persuasive but factual — written to overturn a likely denial.
